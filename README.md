# Управление больницей (Mb)

Учебный проект — настольное приложение для управления данными больницы (отделения, врачи, пользователи). Курсовая работа, 1 курс.

## Стек

- **.NET 9** (Windows Forms)
- **Microsoft Access** (`.accdb`) через **OleDb** (`Microsoft.ACE.OLEDB.12.0`)
- **xUnit** для тестов
- **Microsoft.Extensions.DependencyInjection** для DI

## Структура


## Структура БД

| Таблица | Поля |
|---|---|
| `Больницы` | `Номер_больницы` (PK), `Адрес` |
| `Отделения` | `Код_отделения` (PK), `Название_отделения`, `Номер_больницы` (FK), `Число_мест`, `Полезная_площадь_на_место`, `Число_свободных_мест` |
| `Врачи` | `Код_врача` (PK), `ФИО_врача`, `Что_окончил`, `Должность`, `Специальность`, `Код_отделения` (FK) |
| `Пользователи` | `Код_пользователя` (PK), `Логин` (уникальный), `Пароль` (SHA-256, 64 симв.), `Роль` |

Связи: `Больницы` 1—∞ `Отделения` ∞—1 `Врачи` (с каскадным удалением).

## Роли и права

- **Администратор** — полный доступ ко всем таблицам, единственный, кто может удалять.
- **Главврач** — редактирование таблиц «Врачи» и «Отделения».
- **Сотрудник** — только просмотр.
- **Гость** — выдаётся при регистрации, только просмотр.

## Запуск

1. Установить **Microsoft Access Database Engine 2016 Redistributable** (x64) — нужен для драйвера OleDb.
2. Файл `Hospital.accdb` лежит в корне проекта и автоматически копируется в `bin\Debug\net9.0-windows\` при сборке.
3. Собрать и запустить:
   ```
   dotnet build
   dotnet run --project Mb.csproj
   ```

Стандартный вход: `admin` / `admin` (пароль захеширован SHA-256 в БД).

## Тесты

```
dotnet test
```

11 тестов покрывают:
- хеширование паролей (`PasswordHasher`),
- сессию пользователя (`UserSession`),
- авторизацию (`AuthService`) — с использованием **mock-объекта** `FakeDbManager`, реализующего `IDbManager`. Это позволяет тестировать логику без подключения к реальной БД.

## Безопасность

- Параметризованные SQL-запросы (защита от инъекций).
- Хеширование паролей (SHA-256).
- Защита от брутфорса: 3 неудачные попытки → блокировка на 30 минут.
- Разграничение прав по ролям на уровне сервиса (`DataService.CanEdit`).
